# Claude Code 입력용 PRD: 네이버 가격비교 상위노출 분석 SaaS

아래 요구사항을 기준으로 MVP 수준의 웹서비스를 설계하고 구현해줘.
문서는 제품 요구사항(PRD) + 개발 지시사항을 함께 포함한다.

---

## 1. 제품 한줄 정의

네이버 쇼핑 **가격비교 탭**에서 **광고 상품**과 **브랜드 카탈로그 묶음 상품**을 제외한 실제 일반 셀러 상품만 분석하여,
사용자가 특정 키워드에서 상위노출 가능성이 높은 **상품명 패턴**과 **추천 상품명**을 얻을 수 있게 해주는 SaaS 웹서비스를 만든다.

---

## 2. 해결하려는 문제

현재 사용자는 네이버 가격비교 탭에서 특정 키워드를 검색한 뒤,
상위 상품을 직접 보면서 아래 작업을 수작업으로 정리하고 있다.

- 상위 노출 상품 확인
- 광고 상품 제외
- 브랜드 카탈로그 상품 제외
- 일반 셀러 상품만 따로 정리
- 상품명 구조 분석
- 리뷰/평점/혜택/썸네일 비교
- 상위노출 가능성이 높은 제목 패턴 도출
- 실제 상품명에 반영

이 작업은 반복적이고 비효율적이며, 사람이 직접 할 경우 판단 편차가 크다.

핵심 문제는 단순히 데이터를 수집하는 것이 아니라,
**일반 셀러가 실제로 벤치마킹 가능한 상품만 분리해서 분석하는 것**이다.

---

## 3. 핵심 목표

이 서비스의 목표는 다음과 같다.

1. 사용자가 키워드를 입력하면 네이버 쇼핑 **가격비교 탭** 기준 데이터를 수집한다.
2. **광고 상품**은 자동 제외한다.
3. **브랜드 카탈로그 묶음 상품**은 자동 제외한다.
4. 제외 후 **일반 셀러 상품 기준 실질 순위**를 다시 계산한다.
5. 상위 상품의 상품명 구조와 반복 패턴을 분석한다.
6. 사용자의 상품 정보를 바탕으로 **추천 상품명**을 생성한다.
7. 분석 결과를 저장하고 다시 조회할 수 있게 한다.
8. 향후 구독형 SaaS로 확장 가능한 구조로 설계한다.

---

## 4. 반드시 지켜야 하는 데이터 수집 기준

### 4-1. 데이터 소스
- 네이버 쇼핑 **가격비교 탭**만 사용한다.
- 일반 쇼핑검색 결과 전체를 기준으로 하지 않는다.

### 4-2. 반드시 제외할 대상
다음은 분석 대상에서 제외한다.

1. **광고 상품**
   - 광고 배지 노출 상품
   - Sponsored 라벨 상품
   - 파워링크형 광고 영역
   - 기타 광고 DOM 구조로 판별 가능한 항목

2. **브랜드 카탈로그 묶음 상품**
   - 동일 상품 다수 판매처 묶음 형태
   - 브랜드 공식 카탈로그형 리스트
   - 판매처 확장형 카드 구조
   - 옵션/브랜드 중심 카탈로그형 UI

### 4-3. 분석 대상
- 광고와 브랜드 카탈로그를 제외한 **일반 셀러 상품만 분석 대상**으로 삼는다.
- 최종 순위는 **제외 후 재정렬된 실질 순위** 기준으로 보여준다.

예시:
- 화면상 1~10위 중 광고 2개, 브랜드 카탈로그 3개가 있으면
- 이를 제외한 후 일반 상품 기준 TOP5를 다시 계산한다.

## 6. 핵심 가치 제안

기존 방식은 사람이 직접 가격비교 탭을 보고,
광고와 카탈로그를 빼고,
엑셀에 정리한 뒤,
감으로 패턴을 해석해야 한다.

이 서비스는 다음 가치를 제공해야 한다.

- 가격비교 탭 데이터를 자동 수집
- 광고 자동 제거
- 브랜드 카탈로그 자동 제거
- 일반 셀러 기준 실질 경쟁 구도 제공
- 상위 상품명 패턴 자동 분석
- 실제 적용 가능한 제목 추천

이 제품의 진짜 USP는 다음이다.

> 단순 크롤링이 아니라, **광고와 브랜드 카탈로그를 제거한 뒤 일반 셀러에게 실제 의미 있는 경쟁상품만 보여주는 것**

## 7. 핵심 기능 요구사항

## 기능 1. 키워드 기반 경쟁상품 분석

### 입력
- 사용자가 키워드를 입력한다.
- 예: 식기건조대, 스텐팬, 텀블러

### 동작
1. 네이버 쇼핑 가격비교 탭에서 검색한다.
2. 상위 결과를 수집한다.
3. 광고 여부를 판별한다.
4. 브랜드 카탈로그 여부를 판별한다.
5. 제외 대상을 제거한다.
6. 남은 일반 상품 기준으로 실질 순위를 다시 계산한다.

### 옵션
- 기본 분석 범위: TOP 5
- 확장 옵션: TOP 10, TOP 20

---

## 기능 2. 수집 데이터 구조

각 상품에 대해 최소 아래 항목을 수집하고 저장해줘.

| 항목 | 설명 |
|---|---|
| originalRank | 원본 페이지 노출 순위 |
| effectiveRank | 제외 후 실질 순위 |
| title | 상품명 |
| price | 가격 또는 최저가 |
| reviewCount | 리뷰 수 |
| rating | 평점 |
| sellerCount | 판매처 수 |
| shippingBenefit | 배송혜택 |
| thumbnailUrl | 썸네일 이미지 |
| isAd | 광고 여부 |
| isCatalog | 브랜드 카탈로그 여부 |
| exclusionReason | 제외 사유 |
| productUrl | 상품 링크 |

## 기능 3. 광고 탐지 엔진

광고 상품을 판별하는 로직을 구현해줘.

### 기준 예시
- 광고 배지 텍스트 존재
- Sponsored 라벨 존재
- 파워링크 영역 포함 여부
- 광고 전용 DOM 셀렉터 매칭

### 요구사항
- 판별 결과를 boolean으로 저장
- 화면에 왜 제외되었는지 설명 가능해야 함

---

## 기능 4. 브랜드 카탈로그 탐지 엔진

브랜드 카탈로그 묶음 상품을 판별하는 로직을 구현해줘.

### 기준 예시
- 동일 상품 다수 판매처 묶음 UI
- 브랜드 공식 카탈로그 영역
- 판매처 확장형 카드
- 카탈로그형 구조 DOM 매칭

### 요구사항
- 판별 결과를 boolean으로 저장
- 화면에 제외 사유를 보여줄 수 있어야 함

---

## 기능 5. 상품명 분석 엔진

일반 셀러 상품만 대상으로 상품명을 분석해줘.

### 분석 항목
- 평균 글자수
- 평균 키워드 수
- 숫자 포함 비율
- 브랜드명 포함 비율
- 재질 키워드 포함 비율
- 속성 키워드 포함 비율
- 용도 키워드 포함 비율
- 타겟 키워드 포함 비율
- 자주 쓰는 단어 TOP 20
- 반복 패턴 추출

### 예시 분해
입력 상품명:
- 올스텐 2단 식기건조대 대형 싱크대선반

분석 예시:
- 올스텐 → 재질
- 2단 → 구조/숫자
- 식기건조대 → 메인키워드
- 대형 → 크기
- 싱크대선반 → 서브키워드

### 구현 방향
- 처음에는 규칙 기반 토큰 분류로 MVP 구현
- 이후 AI/NLP 확장 가능하도록 구조화

## 기능 7. 경쟁강도 분석

키워드별 시장 난이도를 분석하는 지표를 제공해줘.

### 지표
| 지표 | 설명 |
|---|---|
| 광고 점유율 | 상단 결과 중 광고 비중 |
| 카탈로그 점유율 | 브랜드 카탈로그 비중 |
| 일반셀러 기회도 | 일반 셀러 진입 가능성 |
| 리뷰 장벽 | 상위권 평균 리뷰 수 |

### 해석 예시
- 광고와 카탈로그 비율이 높으면 일반 셀러 진입이 어려운 키워드로 본다.
- 일반 셀러 비율이 높으면 기회 키워드로 본다.

## 기능 8. AI 상품명 추천

사용자의 상품 정보를 입력받아 추천 상품명을 생성해줘.

### 사용자 입력값
- 내 현재 상품명
- 브랜드명
- 상품 특성
- 재질
- 용도
- 타겟 고객
- 가격대
- 강조 포인트

### 출력 유형
1. SEO형
2. 클릭유도형
3. 전환형
4. 롱테일 우회형

### 추천 로직 조건
- 상위 일반 셀러 상품 패턴을 반영할 것
- 과도하게 브랜드/카탈로그형 키워드에 의존하지 말 것
- 일반 셀러가 진입 어려운 키워드는 경고할 것
- 가능한 경우 롱테일 우회 키워드도 제안할 것

예:
- “텀블러” 키워드가 브랜드 카탈로그 강세라면
- “대용량 텀블러 빨대 스텐 보온” 같은 우회형 제목도 함께 추천

## 14. 가장 중요한 판단 기준

이 프로젝트의 핵심 성공 조건은 아래 3가지다.

1. **광고를 정확히 제외할 수 있는가**
2. **브랜드 카탈로그를 정확히 제외할 수 있는가**
3. **일반 셀러에게 실제 의미 있는 분석 결과를 주는가**

즉, 이 서비스는 단순 크롤러가 아니라
**일반 셀러 관점에서 쓸모 있는 경쟁상품 분석 도구**가 되어야 한다.

## 15. Claude Code에 바로 붙여넣을 최종 입력문

너는 시니어 풀스택 개발자이자 SaaS 프로덕트 엔지니어다.
Next.js, Supabase, Playwright 기반으로 아래 제품을 MVP 수준으로 설계하고 구현해줘.

제품명은 임시로 PriceRank AI로 한다.

목표는 네이버 쇼핑 가격비교 탭에서 사용자가 입력한 키워드를 기준으로 상위 상품을 수집하되,
광고 상품과 브랜드 카탈로그 묶음 상품을 제외하고,
일반 셀러 상품만 재정렬하여 분석하는 웹서비스를 만드는 것이다.

핵심 기능은 다음과 같다.
- 키워드 입력 후 가격비교 탭 크롤링
- 광고 상품 제외
- 브랜드 카탈로그 상품 제외
- 일반 셀러 상품 기준 실질 순위 재계산
- 상품명 구조 분석
- 경쟁강도 분석
- AI 기반 상품명 추천
- 결과 저장 / 재조회
- 관리자 기능 확장 가능 구조

기술 스택은 Next.js + Tailwind + shadcn/ui + Supabase + Playwright를 사용해줘.

먼저 아래 순서로 진행해줘.
1. 폴더 구조 제안
2. DB 스키마 작성
3. 핵심 페이지 설계
4. 크롤링 모듈 구현
5. 분석 로직 구현
6. UI 구현
7. README 작성

코드는 실제 실행 가능한 형태로 작성하고,
환경변수 예시와 로컬 실행 방법도 함께 제공해줘.

특히 가장 중요한 것은 광고 탐지와 브랜드 카탈로그 탐지 로직이다.
이 부분은 추후 룰 수정이 가능하도록 분리된 구조로 설계해줘.
